A03: Dynamics

Introduction

The purpose of this assignment is to acquaint you with the concepts of creating and using a dynamic simulation for answering questions

Instructions

Part 1: Constraints revisited

The purpose of this exercise is to get experience working with constraints in MuJoCo, and how the parameters work. You will create a planar four-bar linkage in mujoco, adapted from the code shared in class.

  1. You should define a total of five links: A, B, C, D, and Ap. A should be grounded and the other four should initially align with the x asis of your A frame.
  2. Ensure your xml is in "degree" mode

  3. Add the following code to the bottom of the xml, right above the </mujoco> line. What happens? Does this look like a four-bar?

    <equality>
    <weld name="weld1" active="true" body1="A" body2="Ap" relpose="0 0 0 1 0 0 0" anchor="0 0 0" />
</equality>

  4. change axisangle="0 1 0 0" to axisangle="0 1 0 -80" for all bodies. Display the pose at t=0.

    1. Report the joint angles at t=0. Is this surprising? Explain why the values are what they are.
    2. Show the plot at t=3
    3. Now describe what happens when you run the code

  5. Finally, revert axisangle="0 1 0 -90" to axisangle="0 1 0 0" for all bodies. After the line that performs a resetData(), insert the following line:

    data.qpos = numpy.array([-math.pi*80/180,-math.pi*80/180,-math.pi*80/180,-math.pi*80/180])

  6. Play with the relpose and anchor terms. What happens when you use:

    1. relpose="1 1 1 1 0 0 0" anchor="1 1 1"
    2. relpose=".5 0 0 1 0 0 0" anchor="0 0 0"
    3. relpose="0 0 0 .866 0 -.5 0" anchor="0 0 0"
    4. relpose="0 0 0 .866 0 -.5 0" anchor=".5 0 0" Supply an image at t=3 for each of these cases, and explain in your own words what you see for each case. Using these examples, please explain what the first three terms of relpose represent, what the last four terms of relpose represent, and what the three terms of anchor represent.

Part 2: Templates

The purpose of this exercise is to learn how to re-run simulations repeatedly with different model parameters. First, study the concept of string formatting in Python.

  • https://docs.python.org/3/library/stdtypes.html#str.format
  • https://docs.python.org/3/library/string.html#formatstrings

Then using the following code as a starting point,

  1. Ensure the following parameters are set
    1. mass of each body is 1
    2. timestep is set to .001
  2. Modify the xml to create a template, so that all the damper and spring values in the model can be replaced by two variables, b and k, respectively.
  3. Generate xml from the template with with k=10 and b=1. Run the simulation and plot the joint positions over 3 seconds.
  4. put the code to define and run a single simulation into a function that takes two parameters: b and k.
  5. Calculate the sum of squared error between the output position and this data over each time step from t=0s to 3s.
  6. Create a for loop that iterates over all \(k=(k_i)_{k_i=1}^{10}\) and \(b=(.1 b_j)_{b_j=1}^{10}\)
  7. Identify the values of b and k that result in the smallest error.
  8. What have we just done by hand? What steps would you have to do to automate this better using built-in functions already provided in Python or its packages?

Part 1: Confirming the Jacobian

Create a two-link robotic arm following the kinematics diagram found in "Assignment 2: Kinematics", Part 3.

  1. Disable gravity and contact.
  2. Define the bodies.
    1. For the geoms, use rectangular boxes to represent each link, with \(l_b=1.2\) and \(l_c=.8\). Set the width and height of each link to .1
    2. For the joints, use pin / revolute / hinge joints using the figure from Assignment 2 (copied above) as your reference for joint axes and reference frames
  3. Define the initial state of the robot as \(\begin{bmatrix}\theta_1 \\ \theta_2\end{bmatrix} =\begin{bmatrix}\frac{\pi}{6} \\ \frac{\pi}{3}\end{bmatrix}\) and \(\begin{bmatrix}\dot\theta_1 \\ \dot\theta_2\end{bmatrix} =\begin{bmatrix}2.2 \\ 3.1\end{bmatrix}\)
  4. Run the simulation for 3 seconds with a .01 s timestep
  5. Plot the position and velocity of your robot's end-effector. This will be made easier by adding an extra, low-mass body to the end of your robot arm and saving its cartesian position over time.
  6. Discuss whether the speed of your end-effector at t=0 is the same as your calculations in assignment 2 predicted from Part 3, question 5. Why is it the same or different?
  7. Fix whatever you need to in your simulation to resolve any inconsistencies, and re-run the code. Now what do you see?
